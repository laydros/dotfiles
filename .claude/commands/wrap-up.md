End-of-session wrap-up: update memory and journal before Jason closes out.

Be selective. The bar for saving is high — most sessions produce nothing worth keeping. A short "nothing new this session" is a perfectly good outcome.

## Memory

Review this conversation for things that would help a *future* conversation. Skim the existing memory files first so you don't duplicate or contradict what's already there.

Save ONLY if it fits one of the four types and isn't already captured:

- **user** — a new fact about Jason's role, knowledge, or preferences
- **feedback** — a correction or a validated non-obvious approach (include **Why:** and **How to apply:**)
- **project** — a decision, deadline, or stakeholder fact not derivable from code/git
- **reference** — a pointer to an external system

Do NOT save:

- Code patterns, file paths, architecture, conventions — derivable from the repo
- Anything already in CLAUDE.md or an existing memory
- Fix recipes or debugging steps — the commit/code is authoritative
- Recaps of what happened this session
- "Trash info" — if you're unsure whether it's worth keeping, it isn't

Propose each candidate save to Jason in one line ("Save as feedback: X — Y/N?") before writing. Don't batch-save without confirmation.

## Journal

If the `mcp__private-journal__process_thoughts` tool is available, ask Jason whether there are notes to record. If he says yes (or names topics), capture genuine insights — architectural reasoning, surprises, things worth remembering — not a session recap. If the tool isn't available, skip silently.

## Output
